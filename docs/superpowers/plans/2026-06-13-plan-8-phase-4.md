# Plan 8 (Phase 4): HealthKit + Cardio + Barcode

> Sub-skill: superpowers:subagent-driven-development or executing-plans.

**Goal:** (1) Pull weight/steps/workouts/HRV/sleep from Apple Health into the existing collections; (2) Log cardio sessions (run/ride/walk/swim/other) on web + mobile; (3) Scan packaged food barcodes via the phone camera and resolve to macros through OpenFoodFacts.

**Architecture decisions:**
- **HealthKit is iOS-only** via `react-native-health` (Expo config plugin). Read-only — we never write back. Imports are idempotent: each HealthKit sample's UUID is the Firestore doc id.
- **Cardio is its own collection** `cardio_logs/{id}`, distinct from strength `workouts/{id}`. Type-tagged (run | ride | walk | swim | other).
- **Barcode → OpenFoodFacts** via a thin backend proxy that returns an `Estimation`-shaped result, slotting into the existing nutrition logging flow with `source: "barcode"`.
- **Native rebuild required**: Adding `react-native-health` + camera permissions changes Info.plist/entitlements — the iOS app needs a fresh `expo run:ios` build after Task 3.

**Existing:** body metrics (Plan 7.5) already accepts weight; HealthKit weight imports go to that collection. Nutrition router already exists; barcode endpoint slots into it. Workouts collection (strength) is untouched.

### Task 1: Backend — Cardio CRUD + Barcode proxy + HealthKit sync (TDD)

**Files:** `backend/app/schemas.py` (extend), `backend/app/services/cardio_service.py`, `backend/app/services/openfoodfacts.py`, `backend/app/services/healthkit_service.py`, `backend/app/routers/cardio.py`, modify `backend/app/routers/nutrition.py` and `main.py`; add `terraform/main/indexes_cardio.tf`.

Schemas:
```python
CardioType = Literal["run", "ride", "walk", "swim", "other"]


class CardioLogCreate(BaseModel):
    date: str = Field(pattern=r"^\d{4}-\d{2}-\d{2}$")
    type: CardioType
    duration_s: int = Field(ge=0)
    distance_m: float = Field(default=0, ge=0)
    avg_hr: int | None = Field(default=None, ge=20, le=240)
    calories: int | None = Field(default=None, ge=0)
    notes: str = ""
    source: Literal["manual", "healthkit"] = "manual"
    external_id: str | None = None   # HealthKit UUID for idempotency


class HealthKitSample(BaseModel):
    """Single normalized sample posted by the mobile app."""
    kind: Literal["weight", "steps", "workout", "hrv", "sleep"]
    external_id: str
    date: str
    started_at: str | None = None     # ISO 8601 UTC
    ended_at: str | None = None
    value: float | None = None        # weight kg / steps count / hrv ms / sleep min
    workout_type: str | None = None   # if kind=workout
    duration_s: int | None = None
    distance_m: float | None = None
    avg_hr: int | None = None
    calories: int | None = None


class HealthKitBatch(BaseModel):
    samples: list[HealthKitSample]
```

`cardio_service.py` — same shape as `body_service.py`: create_log (idempotent by `external_id` if present — query before insert), list_logs(uid, from, to, limit), get/update/delete with user_id isolation.

`openfoodfacts.py`:
- `lookup_barcode(code: str) -> dict | None`: GET `https://world.openfoodfacts.org/api/v2/product/{code}.json?fields=product_name,brands,nutriments,serving_size,quantity`.
- On 404 or `status != 1`: return None.
- Parse `nutriments` (key per 100g: `energy-kcal_100g`, `proteins_100g`, `carbohydrates_100g`, `fat_100g`) into Estimation shape per the serving_size field (default 100g if unknown). Return `{name, serving, macros, confidence: 0.9, source: "openfoodfacts", code}`.
- Wrap in try/except → return None on any error; never raise.

`healthkit_service.py`:
- `ingest_batch(uid, samples: list[dict])` — single batched write splitting by kind:
  - weight → `body_metrics` doc id = external_id, weight_kg=value, source="healthkit"
  - steps → `daily_metrics/{uid}/days/{date}` increment-set (just track for now; chart later)
  - workout → if `workout_type` in {Running, Walking, Cycling, Swimming, Other} → `cardio_logs` doc id = external_id (idempotent); else skip (strength training stays user-entered to keep set/rep fidelity)
  - hrv / sleep → `health_signals/{uid}/{kind}/{external_id}` raw store for future use
- Each subset wrapped in try/except so a partial failure on one kind doesn't block others. Return `{imported: {weight: N, steps: N, workouts: N, hrv: N, sleep: N}, skipped: N}`.

Routers:
- `routers/cardio.py`: standard CRUD `/api/v1/cardio`.
- `routers/healthkit.py`: `POST /api/v1/healthkit/sync` body `HealthKitBatch` → `ingest_batch` → returns counts.
- Modify `routers/nutrition.py`: add `GET /barcode/{code}` (auth required) → calls `openfoodfacts.lookup_barcode` → 404 if None → returns the Estimation dict.

Indexes (`indexes_cardio.tf`): `cardio_logs (user_id ASC, date DESC)`; `(user_id ASC, external_id ASC)` for idempotency lookups.

TDD: schemas, cardio_service CRUD + idempotency, openfoodfacts parser tests with a mocked response, healthkit ingestion split (mock body_service.create + cardio_service.create), router happy/404/dup paths.

Commit per logical chunk; gate `cd backend && pytest -q` + `terraform validate` after each.

### Task 3: Mobile — HealthKit dev-build + sync

**Files:** mobile/package.json (add `react-native-health`), `mobile/app.json` (config plugin + Info.plist usage descriptions + capabilities entitlement), mobile/src/services/healthkit.ts, mobile/app/(tabs)/index.tsx (Health sync card), mobile/app/settings.tsx (manual sync trigger).

`react-native-health` is an unofficial Expo config plugin (`config-plugins/react-native-health` or use the standard auto-linking). Add to app.json:
```json
"ios": {
  "infoPlist": {
    "NSHealthShareUsageDescription": "Read steps, weight, workouts, HRV, and sleep from Apple Health to track your training.",
    "NSHealthUpdateUsageDescription": "We do not write to Apple Health."
  }
},
"plugins": [..., ["react-native-health", { "isClinicalDataEnabled": false }]]
```

`mobile/src/services/healthkit.ts`:
- `init()` → AppleHealthKit.initHealthKit({permissions: { read: [...10 keys...] }}). Returns boolean.
- `pullSince(date: Date)` → reads weight samples (Quantity HKQuantityTypeIdentifierBodyMass), step count daily totals, workouts (HKWorkoutType), HRV (HKQuantityTypeIdentifierHeartRateVariabilitySDNN), sleep analysis (HKCategoryTypeIdentifierSleepAnalysis).
- Normalize each to `HealthKitSample` shape.
- POST to `/api/v1/healthkit/sync` with the batch.
- Store `lastSyncedAt` in SecureStore.

UI:
- Home: small "Apple Health" row with last sync time + "Sync now" button (idle, syncing, error states).
- Settings: explicit "Connect Apple Health" the first time → prompts permission sheet → success → kicks off initial 30-day backfill.

Caveat (binding): HealthKit requires the **iOS app to be rebuilt** because of native module + plist changes. Document this in the commit message; the controller does the rebuild in Task 5.

Commit: "feat(mobile): apple health sync via react-native-health".
