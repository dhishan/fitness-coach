# Plan 7.5: Body Metrics — Daily Weight + Measurements

> Sub-skill: superpowers:subagent-driven-development or executing-plans.

**Goal:** Daily body check-in: weight (required) + optional body fat %, waist, chest, arms, etc. + optional progress photos. Home card with quick-add and trend chart. Goal suggestions auto-pull the latest weight.

**Architecture:**
- One collection `body_metrics/{id}` — one doc per check-in (date + values; can have multiple per day, latest wins).
- Photos use the existing food-photo upload bucket with `progress/{uid}/` prefix; no new infra.
- Goals service: `suggest_goals` reads latest weight if no `bodyweight_kg` passed.
- Plain backend, no LLM.

Schemas:

```python
class BodyMetricCreate(BaseModel):
    date: str = Field(pattern=r"^\d{4}-\d{2}-\d{2}$")
    weight_kg: float = Field(gt=0, le=400)
    body_fat_pct: float | None = Field(default=None, ge=2, le=70)
    waist_cm: float | None = Field(default=None, ge=20, le=300)
    chest_cm: float | None = Field(default=None, ge=20, le=300)
    arm_cm: float | None = Field(default=None, ge=10, le=100)
    thigh_cm: float | None = Field(default=None, ge=10, le=200)
    photo_urls: list[str] = []
    notes: str = ""


class BodyMetricUpdate(BaseModel):
    weight_kg: float | None = Field(default=None, gt=0, le=400)
    body_fat_pct: float | None = None
    waist_cm: float | None = None
    chest_cm: float | None = None
    arm_cm: float | None = None
    thigh_cm: float | None = None
    photo_urls: list[str] | None = None
    notes: str | None = None
```

`body_service.py`:
- `create_metric(uid, payload)` — store with `created_at`, `user_id`.
- `list_metrics(uid, limit=90)` — order by `date desc`, return list.
- `latest_metric(uid)` — first of list_metrics(1) or None.
- `get/update/delete_metric` with user_id isolation (None on cross-user).
- `latest_weight(uid)` — convenience: latest_metric.weight_kg or None.

`routers/body.py`: standard CRUD `/api/v1/body` (GET list with `limit`, POST create, GET/PUT/DELETE by id), all via `asyncio.to_thread`. Include in `main.py`.

`goals_service.suggest_goals`: if `bodyweight_kg is None`, call `body_service.latest_weight(uid)` before building LLM context.

Index: `body_metrics (user_id ASC, date DESC)`.

Tests: schema validation (reject negative weight; allow None optionals), service CRUD + cross-user isolation, router happy/404 paths, goals fallback uses latest weight.
