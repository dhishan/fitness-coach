### Task 1: Backend - templates CRUD

**Files:** Create `backend/app/services/template_service.py`, `backend/app/routers/templates.py`; modify `backend/app/schemas.py`, `backend/app/main.py`; tests `backend/tests/test_templates_api.py`.

Schemas (add):
```python
class TemplateEntry(BaseModel):
    exercise_id: str
    exercise_name: str
    target_sets: int = Field(ge=1, le=20, default=3)
    superset_group: str | None = None


class TemplateCreate(BaseModel):
    name: str = Field(min_length=1, max_length=80)
    entries: list[TemplateEntry] = []


class TemplateUpdate(BaseModel):
    name: str | None = None
    entries: list[TemplateEntry] | None = None
```

Service: collection `workout_templates`, docs `{user_id, name, entries, created_at, updated_at}`. Functions: `list_templates(user_id)` (order by updated_at desc), `get_template(id, user_id)` (None on missing/cross-user), `create_template(user_id, payload)`, `update_template(id, user_id, payload)`, `delete_template(id, user_id)`. Same patterns as workout_service.

Router `/api/v1/templates`: GET ""(list), POST "" (201), GET/{id} (404 on None), PUT/{id}, DELETE/{id} (204/404). All via to_thread, get_current_user.

Tests mirror test_workouts_api.py style (patch service): list/create/404-cross-user/update/delete. TDD. Composite index NOT needed (single-field user_id filter + order by updated_at requires composite: add `workout_templates (user_id ASC, updated_at DESC)` to terraform indexes_workouts.tf... put it in new file `indexes_templates.tf`).

Commit: "feat: workout templates CRUD".
