# Plan 6 (Phase 2): Remote MCP + Cost-Saving Model Router

> REQUIRED SUB-SKILL: superpowers:subagent-driven-development or superpowers:executing-plans.

**Goal:** (1) Remote MCP at `mcp.fitness-tracker.blueelephants.org` exposing workout/dashboard tools to claude.ai / Claude Desktop / Claude Code, always authenticated. (2) Cheap-classifier router cutting per-turn cost by routing simple lookups to a smaller model.

**Architecture:**
- FastMCP sub-app mounted at `/mcp` on the same FastAPI service (pattern: expense-tracker `backend/app/mcp_server.py`). Sub-app session manager wired into the app lifespan.
- **Auth precedence** (RS256 hard-pinned, never trust `alg` header — lessons #1 binding): Cloudflare Access JWT (`Cf-Access-Jwt-Assertion`) → app-issued bearer JWT → dev-only `X-Mcp-User-Id` (ENVIRONMENT=development only). Resolved user id stashed in a ContextVar so tools can read it.
- **Subdomain via a second Cloud Run domain mapping** `mcp.fitness-tracker.blueelephants.org`, Cloudflare DNS record `proxied = true` so Access injects its JWT. Plain API CNAME stays unproxied.
- **Router:** small classifier LLM call decides `simple` vs `complex`; simple goes to `chat_model_cheap` (default `openai/gpt-4o-mini`), complex stays on `chat_model` (gpt-5.5). One usage event records actual model used + accumulated cost.

**Existing:** all chat + tool service functions (`workout_service`, `dashboard_service`, `exercise_service`); FastMCP needs adding.

### Task 2: MCP auth middleware + ContextVar

**Files:** `backend/app/auth/mcp_auth.py`, tests `backend/tests/test_mcp_auth.py`.

- `_current_user_id: ContextVar[str | None] = ContextVar("mcp_user_id", default=None)`.
- `resolve_user_id_from_request(request) -> str` precedence:
  1. `Cf-Access-Jwt-Assertion` header → `verify_cf_access_jwt` → `idinfo["email"]` → look up Firestore user by email → user.id. Cache (email → uid) via lru_cache on a thin sync function.
  2. `Authorization: Bearer <token>` → `verify_access_token` → payload["sub"].
  3. `ENVIRONMENT == "development"` AND `X-Mcp-User-Id` header → that value.
  4. else → raise HTTPException(401).
- ASGI middleware `McpAuthMiddleware` that calls the resolver, sets the ContextVar, emits structured log `mcp.auth source=cf|jwt|dev user_id=...`.
- Tests: each precedence path; no header → 401; dev header outside dev env → 401.
- Commit: "feat: mcp auth middleware with cf access, bearer jwt, dev fallback".

### Task 4: Cost-saving model router

**Files:** `backend/app/chat/router.py` (new), modify `backend/app/chat/generation.py`, settings, tests `backend/tests/test_chat_router.py`.

- Settings: `chat_model_cheap: str = "openai/gpt-4o-mini"`, `chat_router_enabled: bool = True`, `chat_router_model: str = "openai/gpt-4o-mini"`.
- `chat/router.py::classify(messages: list[dict]) -> Literal["simple", "complex"]`: a single-shot LLM call via `llm.complete` with a strict system prompt: "Reply with exactly one word: simple or complex. 'simple' = the user asks for a single fact or recent number that one tool call answers (e.g. 'how many sessions this week', 'what is my best bench'). 'complex' = analysis, recommendation, plan change, or multiple-step reasoning." Fallback on any error: return `"complex"` (favor accuracy over savings).
- `generation.py::generate_turn_sync`: at the top of the loop, when `chat_router_enabled` and this is the first sub-turn AND there is no prior assistant message, call `classify(messages)`; choose `model = settings.chat_model_cheap if "simple" else settings.chat_model`. Pass that `model` to `llm.complete(messages, tools, model=model)` (add a `model:` parameter to `llm.complete` that overrides the settings default). Record the actually-used model in the single end-of-turn `record_usage` call (so the metering doc reflects the cheaper model when applicable). Add a `model_used` field to `finalize_turn` and `usage_events`.
- Tests: classifier returns simple/complex; router-disabled bypasses classify; classify error → complex; generation uses cheap model for "simple" turns; expensive model for "complex"; usage event reflects the actually-used model.
- Commit: "feat: cost-saving model router for chat turns".
